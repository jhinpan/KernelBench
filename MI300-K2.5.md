# MI300X + Kimi K2.5 (SGLang) + Triton Backend

This note captures a concise map of KernelBench and the MI300X/K2.5 ROCm workflow.

## KernelBench map (what is where)

- `KernelBench/` - dataset files (reference problems)
- `src/kernelbench/` - core logic (prompting, eval, timing, compilation)
- `scripts/` - run/eval helpers and entrypoints
- `results/` - baseline timing data
- `runs/` - generated kernels + eval outputs (created at runtime)

Key pipeline:
1) Generate kernels (LLM) -> stored under `runs/<run_name>/`
2) Evaluate kernels (correctness + performance) -> `runs/<run_name>/eval_results.json`
3) Analyze results (fast_p, etc.)

## MI300X / ROCm assumptions

- Local SGLang server is running at `http://127.0.0.1:30000`
- Kimi K2.5 model id: `/data/Kimi-K2.5`
- ROCm arch target: `gfx942` (MI300X)
- Kernel backend: `triton`

## Helper scripts (this branch)

- `scripts/mi300_k2_5_smoke.sh` - verify local server + model
- `scripts/mi300_k2_5_generate.sh` - generate kernels with K2.5
- `scripts/mi300_k2_5_eval.sh` - evaluate kernels on MI300X
- `scripts/mi300_k2_5_all_levels.sh` - loop generate+eval across levels

## Quick start (single level)

```bash
# install deps (pick one)
# uv sync
# OR: pip install -r requirements.txt  (note: CUDA-specific packages may not apply on ROCm)

# 0) sanity check the server
bash scripts/mi300_k2_5_smoke.sh

# 1) generate kernels (Level 1, all problems)
LEVEL=1 RUN_NAME=mi300_k2_5_level1 bash scripts/mi300_k2_5_generate.sh

# 2) evaluate kernels
LEVEL=1 RUN_NAME=mi300_k2_5_level1 NUM_GPU_DEVICES=8 bash scripts/mi300_k2_5_eval.sh
```

## Running a subset

```bash
# generate only problem IDs 1..5
LEVEL=1 RUN_NAME=mi300_k2_5_l1_subset SUBSET_START=1 SUBSET_END=5 bash scripts/mi300_k2_5_generate.sh

# eval only problem IDs 1..5
LEVEL=1 RUN_NAME=mi300_k2_5_l1_subset SUBSET_START=1 SUBSET_END=5 bash scripts/mi300_k2_5_eval.sh
```

## All levels (long run)

```bash
LEVELS="1 2 3 4" RUN_PREFIX=mi300_k2_5 bash scripts/mi300_k2_5_all_levels.sh
```

## Optional: include hardware info in prompts

```bash
INCLUDE_HARDWARE_INFO=true HARDWARE_GPU_NAME=MI300X \
  LEVEL=1 RUN_NAME=mi300_k2_5_level1_hw bash scripts/mi300_k2_5_generate.sh
```

## Notes

- For ROCm, `gfx942` is passed via `gpu_arch=["gfx942"]` in eval. The updated `set_gpu_arch` also sets `TORCH_HIP_ARCH_LIST`.
- Triton backend is required on ROCm; pass `backend=triton` (already default in the MI300 scripts).
- If you want to run a single problem end-to-end, use:
  `scripts/generate_and_eval_single_sample.py` with `server_type=mi300_k2_5`, `backend=triton`, and `gpu_arch=["gfx942"]`.
